Description

This competition requires participants to submit their data in CSV format. Submissions must be formatted according to the specified structure to facilitate accurate analysis. See the table below for column details and example values:

F1 F2 γ1 γ2 δ1 δ2
1 1 0 0 16 -16
0 1 0 0 3 -10
... ... ... ... ... ...

The number of rows should match the number of timesteps, which is 100 seconds multiplied by 24Hz (2400 rows). Each row represents a timestep with the respective values for each variable.

An example submission can be found here. Use it as a guide to ensure your submission is formatted correctly.

Ensure your data is formatted correctly and adheres to the submission guidelines provided to facilitate accurate analysis and fair competition.

Rules

  • Each participant can submit up to 3 entries per day.
  • Submissions must be in CSV format and should not exceed 5MB in size.
  • Data should be anonymized to protect personal information.
  • The decision of the judging panel is final.
